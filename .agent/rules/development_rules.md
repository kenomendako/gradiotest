# 開発ルールと規約

このファイルは、本プロジェクトの開発において遵守すべき厳格なルールを定義します。

## 1. Git規約

### ブランチ戦略
- **重要: `main` ブランチへの直接コミットは禁止です。**
- タスクごとに必ず新しいブランチを作成してください。
- **ブランチ名の命名規則:** `[prefix]/[task-name-kebab-case]`
    - `feat/`: 新機能
    - `fix/`: バグ修正
    - `improve/`: 改善、リファクタリング
    - `docs/`: ドキュメントのみの変更

### コミットメッセージ
- **言語:** 日本語
- **フォーマット:** `[prefix]: [変更内容の簡潔な説明]`
    - 例: `feat: ログイン画面のデザインを更新`
    - 例: `fix: メモリ検索の重複バグを修正`

## 2. ディレクトリ構成とファイル配置

ドキュメントと計画については、以下の構成に従ってください:

- **INBOX:** `docs/INBOX.md` (一時的なアイデア置き場)
- **タスクリスト:** `docs/plans/TASK_LIST.md` (優先順位付きバックログ)
- **実装計画:**
    - アクティブな計画にはAntigravityの `implementation_plan.md` アーティファクトを使用してください。
- **レポート:** `docs/reports/YYYY-MM-DD_[TaskName].md` (完了レポート)
- **設計判断:** `docs/decisions/NNN_[Title].md` (アーキテクチャ上の決定)
- **ガイド:** `docs/guides/` (仕様書、マニュアル)
- **UIパターン:** `docs/guides/UI_IMPLEMENTATION_PATTERNS.md`

## 3. UI開発ガイドライン

- **パターンファイル:** 一般的なUIコンポーネントやフローについては `docs/guides/UI_IMPLEMENTATION_PATTERNS.md` を参照してください。
- **長時間処理:** ユーザーにフィードバックを提供するため、ジェネレータパターン（`*_with_progress`）を使用してください。
- **状態保持:** 複雑なデータ入力には、中間保存パターンの採用を検討してください。

## 4. 変更履歴 (CHANGELOG)

- **ファイル:** `CHANGELOG.md`
- 更新は以下のカテゴリに分類してください:
    - `Added` (追加)
    - `Fixed` (修正)
    - `Changed` (変更)
    - `Removed` (削除)
