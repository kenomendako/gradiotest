# Documentation Rules

ドキュメントの運用ルールです。

## 1. 完了レポート (Task Report)
タスク完了時には `/task-report` ワークフローに従い、以下の内容を含むレポートを作成してください。
*   問題の概要と背景
*   具体的な修正内容
*   検証結果
*   残課題

## 2. 変更履歴 (CHANGELOG)
*   **ファイル**: `CHANGELOG.md`
*   タスク完了時には必ず更新してください。
*   フォーマット: `YYYY-MM-DD: [変更内容]`

## 3. プロジェクト状態 (STATUS)
*   **ファイル**: `docs/STATUS.md`
*   プロジェクトの現状、達成事項、次のマイルストーンを反映して更新してください。

## 4. 設計判断 (ADR)
重要な技術的決定を行った場合は、`docs/decisions/` に記録を作成してください。
