# タスク整理ワークフロー

INBOXを整理し、マスターリスト内のタスクに優先順位を付けるためのワークフローです。

1. **状況の確認 (INBOX / TASK_LIST / STATUS)**
   - `docs/INBOX.md` を読み、未整理の新規タスクを確認します。
   - `docs/plans/TASK_LIST.md` を読み、現在の優先順位を確認します。
   - `docs/STATUS.md` を読み、現在の目標とロードマップを確認します。

2. **分類と優先順位付け**
   - INBOX内の各アイテムについて、優先度（クリティカル、高、中、低）を提案します。
   - 優先度が不明確な場合は、ユーザーに相談してください。

3. **ファイルの更新**
   - フォーマットされたタスクを `docs/plans/TASK_LIST.md` の適切な優先度セクションに追加します。
   - 移動したアイテムを `docs/INBOX.md` から削除します。
   - `docs/STATUS.md` の「ロードマップ（次の予定）」を最新の優先順位に合わせて更新します。
   - `docs/INBOX.md` をクリーンな状態にします（意図的に残すアイテムがある場合を除く）。
