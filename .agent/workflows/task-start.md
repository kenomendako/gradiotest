# タスク開始ワークフロー

計画完了後、実装のための環境をセットアップするワークフローです。

1. **前提条件の確認**
   - `implementation_plan.md` が存在し、承認されていることを確認します（このワークフローが呼ばれた時点で承認済みとみなします）。

2. **Gitブランチの作成**
   - タスクの種類と名前に基づいてブランチ名を決定します（例: `feat/new-ui`, `fix/bug-123`）。
   - 以下のコマンドを実行します（自動実行可）:
   // turbo
   ```bash
   git checkout -b [branch-name]
   ```

3. **ブランチの確認**
   - `git branch` を実行し、ブランチが切り替わったことを確認します。

4. **準備完了**
   - 実装を開始することを宣言します。
