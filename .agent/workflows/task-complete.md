# タスク完了ワークフロー

実装と検証が完了し、`task-report` での報告と承認が済んだ後に実行します。

1. **メインブランチへの切り替え**
   - 現在のブランチでの作業が全てコミットされていることを確認してください。
   - `git checkout main`

2. **最新状態の取得**
   - リモートの変更を取り込みます。
   - `git pull origin main`

3. **機能ブランチのマージ**
   - 作業していたブランチをマージします。
   - `git merge [branch-name]`
   - ※ コンフリクトが発生した場合は解消し、コミットしてください。

4. **変更の最終確認**
   - `CHANGELOG.md` と `docs/STATUS.md` が正しく更新されているか最終確認します。

5. **リモートへのプッシュ**
   - 変更をリモートリポジトリに反映します。
   - `git push origin main`

5. **ブランチのクリーンアップ**
   - 不要になったローカルブランチを削除します。
   - `git branch -d [branch-name]`

6. **完了確認**
   - `/task-grooming` を実行し、次のタスクに着手するか、INBOXを確認してください。
