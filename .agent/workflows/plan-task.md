# タスク計画ワークフロー

特定のタスクに対する計画フェーズを開始するためのワークフローです。

1. **タスクの選択**
   - `docs/plans/TASK_LIST.md` を読みます。
   - 未着手のうち、最も優先度の高いタスクを特定します。
   - そのタスクに取り組むかどうか、ユーザーに確認します。

2. **進行中としてマーク**
   - `docs/plans/TASK_LIST.md` を更新し、タスクを `[/]` (進行中) にマークします。

3. **実装計画の作成**
   - `implementation_plan.md` アーティファクトを日本語で作成または更新します。
   - **ゴール:** タスクの説明をコピーします。
   - **提案される変更:** 修正するファイルをリストアップします。
   - **検証計画:** テスト方法を定義します。

4. **レビュー**
   - `notify_user` を使用して、計画のレビューをユーザーに日本語で依頼します。
