# 開発サイクルワークフロー `/dev-cycle`

INBOXの思いつきタスクを整理し、優先順位をつけて実装、完了レポートまで一貫して管理する標準ワークフロー。

---

## フェーズ1: タスク整理

### 1.1 INBOX確認
`docs/INBOX.md` の未整理タスクを確認する。

### 1.2 タスクリスト更新
`docs/plans/TASK_LIST.md` に優先度順で追加する。

**優先度ラベル:**
- 🔴 クリティカル: リリースブロッカー
- 🟠 高: ユーザー体験に大きく影響
- 🟡 中: リリース後でも可
- 🟢 低: 将来機能

### 1.3 INBOX整理
タスクリストに移行したタスクは `docs/INBOX.md` から削除する。
（INBOXは常にクリーンに保つ）

---

## フェーズ2: タスク計画

### 2.1 次のタスク選定
`docs/plans/TASK_LIST.md` から優先度の高いタスクを選ぶ。
タスクを `[/]` に変更して作業中であることを示す。

### 2.2 計画書作成
Antigravityのブレインフォルダに `implementation_plan.md` を作成。
（Antigravityの標準機能を使用）

### 2.3 ユーザー承認
計画書をユーザーに提示し、承認を得る。

---

## フェーズ3: 実装

### 3.1 ブランチ作成
// turbo
```bash
git checkout -b [prefix]/[task-name]
```

**prefixの選び方:**
- `fix/` - バグ修正
- `feat/` - 新機能
- `improve/` - 改善・リファクタリング
- `docs/` - ドキュメントのみ

### 3.2 実装作業
計画書に従ってコードを実装する。

### 3.3 記録作成（必要に応じて）

**設計判断記録 (`docs/decisions/`)**:
- ファイル名: `NNN_[判断名].md`（連番）
- **記録すべきケース:**
  - 複数の選択肢から一つを選んだ場合
  - 技術的制約で機能を断念した場合
  - 将来の方針に影響する決定をした場合
- 形式: 背景、選択肢、決定、理由、影響を記載

**教訓・知見**: `docs/journals/[カテゴリ]_LESSONS.md` に追記

**仕様変更**: `docs/guides/` の関連ファイルを更新

---

## フェーズ4: 完了処理

### 4.1 動作確認
アプリを起動して変更内容を検証する。

### 4.2 レポート作成
`docs/reports/YYYY-MM-DD_[タスク名].md` に完了レポートを作成。

**テンプレート:** `docs/templates/report_template.md` をコピーして使用

**レポートに含める内容:**
- 問題の概要
- 修正内容
- 変更したファイル
- 検証結果
- 残課題（あれば）

### 4.3 コミット
```bash
git add .
git commit -m "[prefix]: [変更内容の簡潔な説明]"
```

### 4.4 マージ & プッシュ
動作チェックと承認があってから行う。
**特にメインブランチへのマージ前には必ずユーザーへの確認を取ること。**
```bash
git checkout main
git pull origin main
git merge [ブランチ名]
git push origin main
```

### 4.5 CHANGELOG更新
`CHANGELOG.md` に変更内容を追記する。

**セクション:**
- `Added` - 新機能
- `Fixed` - バグ修正
- `Changed` - 仕様変更
- `Removed` - 削除された機能

### 4.6 タスクリスト更新
`docs/plans/TASK_LIST.md` の該当タスクを `[x]` に変更し、
レポートへのリンクを追記する。

### 4.7 ブランチ削除（任意）
```bash
git branch -d [ブランチ名]
```

## クイックリファレンス

| ファイル | 用途 |
|----------|------|
| `docs/STATUS.md` | プロジェクト全体の現在状態サマリー |
| `docs/INBOX.md` | 思いつきタスクの一時置き場 |
| `docs/plans/TASK_LIST.md` | 優先度順のマスタータスクリスト |
| `docs/templates/` | レポート・設計判断のテンプレート |
| `docs/decisions/` | 設計判断の記録 |
| `docs/guides/` | 仕様書・指示書 |
| `docs/journals/` | 教訓・知見の蓄積 |
| `docs/reports/` | 完了レポート |
