# 開発サイクルワークフロー `/dev-cycle`

これは開発サイクル全体のマスターガイドです。
具体的な作業フェーズに応じて、以下のサブワークフロー（スラッシュコマンド）を使用してください。

## 1. タスク整理 (Phase 1)
**コマンド:** `/task-grooming`
- **目的:** `INBOX` のアイデアを整理し、優先順位をつけて `TASK_LIST` に移動します。
- **タイミング:** 開発を始める前、またはINBOXが溜まった時。

## 2. タスク計画 (Phase 2)
**コマンド:** `/plan-task`
- **目的:** 取り組むタスクを決定し、`implementation_plan.md` を作成します。
- **タイミング:** 具体的な実装作業に入る前。

## 3. 実装開始 (Phase 3)
**コマンド:** `/task-start`
- **目的:** 作業用ブランチを作成し、コーディングを開始します。
- **タイミング:** 計画が承認された直後。

## 4. 完了報告 (Phase 4)
**コマンド:** `/task-report`
- **目的:** 作業内容をまとめ、レポート作成、コミット、CHANGELOG更新を行います。
- **タイミング:** 実装と動作確認が完了した後。
