## 未整理タスク

- [ ] [ログ修正機能で修正されないことや思考ログが消えることがある]
  - 詳細: 望む挙動は、思考ログ内も読点修正。
  - 優先度: 🔴高 

- [ ] [内部仕様モデルのカスタム設定]
  - 詳細: 内部の高速モデルや要約モデルを今はGeminiで固定しているが、ユーザーが好みのモデルに変更できるようにする。GLM-4.7などを検討する。
  - 優先度: 🔴高 

 - [ ] [主観的日記・創作ノート・研究ノートのRAWエディタがまたスクロールしなくなってる]
  - 詳細: 研究ノートは罫線のせいで非等の内容が「研究ノート」という見出しだけのエントリと本分との二つに分かれてしまってるので、記述方法かエントリの表示の分け方を見直す。
  - 優先度: 🔴高

- [ ] [アバター表情のフォールバック変更]
  - 詳細: 今はキーワードでフォールバックしているが、ペルソナ感情でフォールバックするようにする。デフォルトの表情パターンはペルソナ感情と同じだけ設定する。従来通り、各ルームのシステムプロンプトで独自の表情パターンに上書き可能に。
  - 優先度: 🟡中

- [ ] [自律行動時・通常応答時だけでなく、アラームの時など、いつであってもペルソナのツール使用はログに記録する]
  - 詳細: 透明性と前回何をしたかのペルソナの記憶のため
  - 優先度: 🔴高

- [ ] [エピソード記憶の分量・質・学習メカニズムの再検討]
  - 詳細: 1日分の要約が長すぎる（1000-3000文字）。MemRL/GDPOの概念を取り入れ、Arousal（感情の振れ幅）ベースの取捨選択基準を検討。
  - 研究メモ: [episodic_memory_memrl_study.md](plans/research/episodic_memory_memrl_study.md)
  - 優先度: 🔴高

- [ ] [スーパーツール化によるトークン削減]
  - 詳細: LangChainがツールをモデルにバインドする際、各ツールのJSONスキーマ（約400トークン/ツール、合計約12kトークン）が送信される。関連する複数のツールを1つの「スーパーツール」に統合し、operationパラメータで機能を切り替えることで、ツール数を削減しトークン消費を抑える。
  - 候補: 記憶系ツール（read/write/search/list）、ノート系ツール（日記/創作/研究）、タイマー系ツール
  - 優先度: 🟡中
