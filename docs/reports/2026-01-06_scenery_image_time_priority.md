# 情景画像表示ロジック見直し

**完了日**: 2026-01-06  
**ブランチ**: `improve/scenery-image-time-priority`

## 問題概要
情景画像のフォールバック検索において、「冬の昼」の画像がない場合に「冬の夜」が表示されてしまう問題があった。ユーザーの期待は「季節を変えても同じ時間帯（秋の昼など）」を表示すること。

## 修正内容

### 変更ファイル
- [utils.py](file:///c:/Users/baken/OneDrive/デスクトップ/gradio_github/gradiotest/utils.py) の `find_scenery_image` 関数 (v3 → v4)

### 変更後の検索優先順位
1. `場所_現在季節_時間帯.png` (完全一致)
2. `場所_[他の季節]_時間帯.png` (同じ時間帯で季節を遡る)
3. `場所_時間帯.png` (時間帯のみ)
4. `場所_季節.png` (季節のみ)
5. `場所.png` (デフォルト)

### 季節フォールバック順序
現在季節から逆順に遡る:
- 冬 → 秋 → 夏 → 春
- 秋 → 夏 → 春 → 冬
- 夏 → 春 → 冬 → 秋
- 春 → 冬 → 秋 → 夏
