# 設計判断記録

このディレクトリには、プロジェクトの重要な設計判断を記録します。

## なぜこのディレクトリがあるか

- 将来「なぜこうなっているのか？」を防ぐ
- 同じ議論を繰り返さない
- 新しい開発者（AI含む）が文脈を理解できる

## 記録の書き方

```markdown
# [タイトル]

**日付**: YYYY-MM-DD  
**ステータス**: 承認済み / 検討中 / 却下

## 背景
なぜこの判断が必要だったか

## 選択肢
1. 案A: ...
2. 案B: ...

## 決定
選んだ案とその理由

## 影響
この判断により何が変わるか
```
